System 01 — AI-First Cyber Threat Intelligence Pipeline

Type: Production pipeline · Pattern: ingestion → enrichment → storage → consumers · Primary user: analysts + downstream automation

Problem

Convert a high volume of heterogeneous cybersecurity content (feeds, reports, advisories, blogs) into reliable, structured intelligence that analysts can trust and that automation can consume. The challenge is not summarization — it’s precision, consistency, and operational scalability.

Constraints

  • Correctness & trust: avoid generic or invented entities; outputs must be traceable.
  • Cost control: LLM calls are budgeted; avoid wasting tokens on low-signal content.
  • Throughput: continuous ingestion with batching, retries, and backpressure.
  • Operational maintenance: logging, run artifacts, reproducibility, and monitoring are non-negotiable.

Architecture (high level)

Core idea: schema-first enrichment + multi-stage filtering + AI as bounded tools.

Ingestion
RSS / URLs
Deduping, normalization, source scoring
Triage
Signal gating
Skip low-value content before AI spend
Enrichment
LLM + rules
Entities, CVEs, IOCs, taxonomy, titles
Storage
DB + vector
Structured store + retrieval layer
Consumers
Dashboards
Analyst UI, reports, search, exports
Ops
Artifacts
Logs, run IDs, CSV/JSON outputs, metrics

Key decisions

  • Schema-first extraction with strict JSON constraints and enumerations for high precision.
  • Multi-stage gating to minimize AI spend: classify relevance before enrichment.
  • Guardrails & consistency rules (e.g., “no generic entities”, mandatory victim extraction, etc.).
  • End-to-end traceability: store prompts/versions, outputs, and run artifacts for auditability.
  • Cost accounting: token/cost tracking per model to keep the system sustainable.

Outcomes

  • Reliable entity-rich intelligence with an analyst-editable workflow.
  • Scalable pipeline with reproducible outputs (run IDs, artifacts).
  • Operational control over quality, cost, and long-term maintainability.

Next

  • Formalize evaluation suites (precision/recall per entity type, drift tracking).
  • Add automated regression testing for prompts and rule changes.
  • Expand “original source detection” and multi-source attribution.
